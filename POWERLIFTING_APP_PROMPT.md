# 力量举训练计划规划与记录小程序需求 Prompt

你是一名专业的微信小程序开发工程师。请在当前微信小程序项目中持续开发一款用于力量举训练计划规划与训练记录的小程序，遵循现有原生小程序 TS/LESS 结构，优先保持实现简洁、可靠、可扩展。

## 产品定位

这是一款面向力量举训练者的本地训练日志与计划工具。核心目标是让用户快速记录卧推、深蹲、硬拉等训练内容，并能够长期保存、查看、导入和导出自己的训练数据。

## 设计风格

- 整体 UI 走优雅、简洁、克制的风格。
- 信息层级清晰，适合高频记录和回看。
- 避免营销式落地页，打开后直接进入可用的训练记录体验。
- 交互控件需要稳定、易点按，适配手机屏幕。

## 初版功能范围

### 1. 训练记录

- 展示所有训练记录。
- 默认按训练日期由近到远排序。
- 支持切换到日历模式，查看哪些日期有训练记录。
- 日历中有训练的日期需要有明确视觉标记。

### 2. 创建新训练记录

- 用户可以选择训练日期。
- 用户可以选择训练动作，例如卧推、深蹲、硬拉。
- 每个动作可以添加多组训练数据。
- 每组训练数据包含：次数、RPE 值，RPE 范围为 1-10。
- 支持添加新的训练动作。
- 支持在一次训练记录中添加多个动作。
- 动作选择不要使用普通 select。因为未来每个动作会加配图或更多说明，动作选择应使用弹窗、抽屉、卡片列表或独立页面，方便扩展展示内容。

### 3. 关于

- 关于页用于介绍本应用，初版可先保留简短占位内容。
- 关于页提供训练记录导出能力。
- 关于页提供训练记录导入能力。

## 数据存储与导入导出

- 训练记录必须长期存储在微信小程序本地存储中。
- 导出应生成 JSON 文件，方便用户备份到手机或分享保存。
- 导入应支持一次选择多个文件。
- 导入成功后直接覆盖当前本地训练记录。
- 因为导入会覆盖数据，执行导入前必须弹窗提醒用户确认。
- 导入文件格式应优先兼容本应用导出的 JSON。

## 后续扩展方向

- 为每个训练动作增加配图、动作说明和技术要点。
- 增加训练计划模板和周期规划。
- 增加重量、容量、估算 1RM、趋势图等统计能力。
- 增加按动作筛选训练历史。
- 增加数据格式版本迁移，确保未来导入旧备份仍可用。

## 开发要求

- 保持代码风格与当前项目一致。
- 不引入不必要的依赖。
- 功能改动应尽量集中且可验证。
- 本地存储 key、导出数据结构和类型定义需要清晰，方便后续维护。
